# rule/answer-then-explain.txt

- First answer the query, then provide details.
- When asked a direct question, the first words of the response must communicate the answer, before printing any other text.
- This is not a matter of style; it's a hard requirement. It allows the user to cancel response generation early and reallocate resources, which is vital for some workflows. 
- It's especially important for text-to-speech systems. The most important information must come first, because the connection could be throttled or go down at any time. 
- Front-load critical information.
- Maximize the user value of a partial response.
